作用域、闭包

作用域

定义

代码在运行时，各个变量、函数和对象的可访问性。换句话说，作用域决定了你的代码里的变量和其他资源在各个区域中的可见性

分类

1､ 全局作用域 （在函数之外的变量）

当你在文档中（document）编写 JavaScript 时，你就已经在全局作用域中了。JavaScript 文档中（document）只有一个全局作用域。定义在函数之外的变量会被保存在全局作用域中

var name = ‘hello world’;

// the scope is by default global

2､ 局部作用域 （在一个函数中的变量）

在函数中的变量就在局部作用域中。并且函数在每次调用时都有一个不同的作用域。这意味着同名变量可以用在不同的函数中。因为这些变量绑定在不同的函数中，拥有不同作用域，彼此之间不能访问。

每个函数在调用的时候会创建一个新的作用域。

全局作用域

当你在文档中（document）编写 JavaScript 时，你就已经在全局作用域中了。JavaScript 文档中（document）只有一个全局作用域。定义在函数之外的变量会被保存在全局作用域中

定义

一个函数连同它的词法作用域所在的，使用的这个变量的集合

讲真，其实单看这定义，很难理解闭包，我们来看实例吧

实例

对应的是一个数组，数组里面存的都是函数,函数里有一个Scopes，Scopes里对应的是一个全局作用域，为什么呢？

作用域链是指当我们去使用一个变量时，会从它的一个，就是这个函数内部去找。其实就是函数内部的一个活动对象AO，如果找不到，再从它的一个词法作用域去找，它的词法作用域是什么呢，就是这个函数所创建的所得的作用域

所以的话，打开这个函数会有一个Scopes

作用域链是指一个函数内部，当我们去使用一个变量的时候，这个变量从哪去找？

如果这个函数内部有声明，当然我们从函数内部去找，如果找不到的话，再从函数外部去找

那函数外部是指什么呢？

函数外部是指这个函数在声明的时侯，所在的这个作用域，也就是它的词法作用域，换句话说，也就是创建当前函数所在的作用域，也就是写它的时候所在的作用域。从这里面找，如果再找不到，从上面的函数的词法作用域去找，一直找，找到最后，如果真找不到，那就是undefined,或者是直接报错

词法作用域 (lexical environment)

  • 函数在执行的过程中，先从自己内部找变量
  • 如果找不到，再从创建当前函数所在的作用域(词法作用域)去找, 以此往上
  • 注意找的是变量的当前的状态

函数连同它作用域链上的要找的这个变量，共同构成闭包，换句话说，闭包就是一个变量，这个变量临时去存我们的数据,一般来说，它和一个函数同时存在

一般情况下使用闭包主要是为了

  1. 封装数据
  2. 暂存数据

来个典型的闭包实例

分析：

有一个函数car ,函数里有一个变量speed,函数内部又声明了一个函数fn，然后return fn

假设car 里面没有fn 函数这一堆东西，当我执行 var speedUp = car(),执行完成之后，那car函数内部的局部变量就会消毁掉，那么speed 就没了，就不存在了

那现在我们在函数car 内部声明了fn 函数，把这个函数返回出来return fn，赋值给speedUP,这个时候，因为我们的全局作用域是一直存在的，除非把当前页面给关掉，它才被消毁，所以的话，它一直存在的话，这个speedUp 就是一直存在的

而这个speedUP是指什么呢，也就是指这个fn，也就是car函数内部声明的这个fn函数

那这个fn函数就是一直存在,换句话说，这个car 函数作用域内的东西就无法被销毁，也就是变量speed和fn函数无法销毁,因为它里面有东西被人用，和浏览器的垃圾回收机制是相关的

当我们去执行speedUp（）时，实际上就是去执行fn函数,然后fn函数内部是没有声明这个speed变量的，它会从上一级的词法作用域去找，也就是这个函数fn,声明时所在的作用域，也就是car函数内部, 最后找到这个speed,然后把这个值speed++ ,即这个speed 值加1,然后再次调用这个函数，所以又在之前的值基础上，再加1

换句话说，正常情况下，car 执行完后，speed变量就消失了，就没有了，我们也无法去获取，但是现在在函数car内部，又返回出一个函数fn,然后函数fn内部又使用这个speed, 就导致创建了一个闭包

那什么是个闭包呢

其实就是这个函数fn连同它词法作用域下所在的使用的这个变量speed的集合

也就是变量speed 和 函数fn 加在一起，就是一个闭包

如果car 函数内 不return fn ，那这里面还有没有闭包呢？

如果不执行return fn的话，那car函数 执行完之后，fn函数内的所有的局部变量都会销毁掉,销毁掉就什么都没有了，什么都没有了就不存在有没有闭包了

闭包有什么用呢？我们直观的看一看

1､当我调用speedUp()时，fn函数内的speed 的值会加1,换句话说这个变量是不是就暂存下来了，我们无法去直接访问这个变量speed,但是我们可以去操作它，而且呢，它会暂存起来，不会被销毁掉

2､第二个就是封装数据，就是把speed 给封装起来

首先这里有一个空数组 fnArr,然后我们去遍历这个数组,遍历时侯给这个数组去赋值

那这个时侯console.log( fnArr[3]（）），那fnArr[3] 是什么呢，它里面也是存的一个函数

函数后面加一个括号 （） ，就可以去执行, 执行的时候问你，执行的结果是多少？
